# 「입양 후 케어」 탭 화면 기획서

> 입력 데이터: `pet_insurance.json`(보험 5개사) + `커머스_제휴처_조사.md`(품목 10개·제휴처 6곳)
> 상위 문서: 「BM_강화안.md」 — 원칙: 입양 전 무료, 수익화는 이 탭에서
> MVP 구현 수준: **정적 페이지 1장** (API·DB 연동 없음, 데이터는 JSON 하드코딩)
> 작성일: 2026-07-02

---

## 1. 이 화면의 역할

케어 탭은 "입양 문의를 마친(또는 입양한) 사용자가 다음에 할 일"을 안내하는 화면입니다. 사용자에게는 **입양 준비 도우미**, 우리에게는 **수익이 발생하는 유일한 화면**입니다. 광고판이 아니라 체크리스트처럼 보여야 이 두 역할이 동시에 성립합니다.

**진입 경로 (2곳)**

1. 입양 문의 안내 화면 하단 → "입양이 결정됐다면? 준비를 도와드릴게요" CTA
2. 글로벌 내비게이션(헤더) → "입양 후 케어" 탭

---

## 2. 화면 구조 — 위에서 아래로 4블록

조사 문서의 MVP 우선순위(① 쿠팡 용품 → ② 훈련 → ③ 건강검진)를 반영하되, **화면 배치는 사용자 여정 순서**(짐 챙기기 → 보험 → 전문가)로 정렬합니다. 수익 기여가 큰 커머스가 자연스럽게 최상단에 옵니다.

```
┌─────────────────────────────────────────────┐
│ 헤더: "○○이와의 새 생활, 준비를 도와드릴게요"   │  ← 매칭했던 동물 이름 삽입(있으면)
├─────────────────────────────────────────────┤
│ 블록 A. 입양 준비물 체크리스트  (10개 항목)      │  ← 커머스 (쿠팡 파트너스)
│   ☐ 이동장(켄넬) — 입양 당일 안전 이동에 필수    │
│      [준비하러 가기 →]                         │
│   ☐ 자동 급식기 🔶추천 — 부재 시간이 길어서...   │  ← 진단 입력값 기반 "추천" 배지
│   ... (총 10개, 체크박스 + 이유 + 링크 버튼)     │
├─────────────────────────────────────────────┤
│ 블록 B. 우리 아이 맞춤 펫보험  (카드 3~5장)      │  ← 보험 제휴
│   [로고] 삼성화재 다이렉트 착한펫보험            │
│   월 29,827원~ · 치료비 70% 보장               │
│   "동물등록 없이 사진 1장으로 가입 가능"          │
│   [자세히 보기 →]          (배지: 입양 할인 3%) │
├─────────────────────────────────────────────┤
│ 블록 C. 전문가 연결  (카드 2~3장)               │  ← 리드 제너레이션
│   행동교정·훈련 / 건강검진 키트                  │
├─────────────────────────────────────────────┤
│ 블록 D. 적응 가이드 (기존 Could 콘텐츠, 텍스트)  │  ← 파양 방지 콘텐츠, 수익 없음
│   "첫 7일은 이렇게 하세요"                      │
└─────────────────────────────────────────────┘
하단 고지: 제휴 링크 안내 문구 (7절 참고)
```

## 3. 블록 A — 입양 준비물 체크리스트 (커머스)

**UI 형태**: 체크박스 리스트 (카드 그리드 아님). "쇼핑 목록"이 아니라 "준비 목록"으로 보이게.

| 화면 요소 | 데이터 출처 (커머스 조사 부록 JSON) |
|---|---|
| 항목명 | `items[].name` |
| 한 줄 이유 | `items[].why` |
| [준비하러 가기] 버튼 | `items[].coupang_url` (→ 파트너스 추적링크로 교체) |
| 🔶 추천 배지 | 아래 개인화 규칙 |

**개인화 규칙 (Should — 진단 입력값 재사용)**

조사 문서의 연동 아이디어를 규칙표로 확정합니다. 적합 판단 때 받은 6개 입력값을 그대로 쓰므로 추가 질문이 없습니다.

| 진단 입력값 | 조건 | 동작 |
|---|---|---|
| 하루 외출 시간 | 8시간 이상 | ⑦ 자동 급식기, ⑧ 펫캠에 "추천" 배지 + 리스트 상단 이동 |
| 산책 가능 시간 | 1시간 이상 | ⑥ 리드줄·하네스에 "추천" 배지 |
| 활동성 선호 | 활발한 아이 | ⑨ 노즈워크 장난감에 "추천" 배지 |
| 반려동물 경험 | 없음 | ⑤ 배변패드, ⑩ 방석·하우스에 "첫 입양 필수" 배지 |

배지 문구는 이유와 함께: "외출 시간이 길다고 하셨죠? 정시에 밥을 챙겨줘요" — 톤 가이드(생활 맥락 번역) 원칙 유지.

**MVP 단순화**: 개인화가 Day 4까지 안 되면 배지 없이 10개 고정 리스트로 출시. 배지는 하드코딩 데모(오소리 페르소나 = 8시간 부재 기준)로만 시연해도 효과 충분.

## 4. 블록 B — 맞춤 펫보험 (핵심 수익모델)

**UI 형태**: 가로 스크롤 카드 3~5장. 정렬 기준: **① 입양 할인 있는 곳 우선 → ② 월 보험료 낮은 순.**

정렬 결과(현재 데이터 기준): DB(33,560원·할인3%) → 현대(39,710원·할인3%) → KB(40,061원·할인3%) → 삼성(29,827원·할인없음) → 메리츠(36,820원·할인없음)

> 단, 삼성이 최저가이므로 "최저가" 배지를 따로 부여해 카드에 노출. 사용자가 "할인"과 "저렴함"을 다 볼 수 있게.

| 화면 요소 | 데이터 출처 (`pet_insurance.json`) | 표시 예 |
|---|---|---|
| 로고 | `insurers[].logo_url` | (파비콘, 정식 배포 시 교체) |
| 회사·상품명 | `name` + `product` | DB손해보험 펫블리 반려견보험 |
| 월 보험료 | `monthly_premium_krw` | "월 33,560원~" (물결 필수 — 조건별 상이) |
| 핵심 배지 | `adoption_discount_percent` | 🏷 "유기동물 입양 할인 3%" |
| 보장 요약 | `coverage_summary` 1줄 축약 | "치과·피부·관절 + MRI/CT 보장" |
| 유기동물 안심 문구 | `abandoned_note` | "동물등록 없어도 사진 1장으로 가입" |
| CTA | `source_url` | [자세히 보기 →] |

**카드 위에 놓을 안심 헤드라인 (이번 조사의 최대 수확)**

> "유기동물이라 보험이 안 될까 걱정하셨나요? **주요 5개사 모두 가입할 수 있어요.** 3곳은 입양 할인까지 있어요."

이 문구는 조사 `meta.conclusion`을 그대로 사용자 언어로 번역한 것으로, 입양 꺼리는 이유 2위(질병 걱정 30.1%)를 정면으로 해소하는 카피입니다.

**개인화 규칙 (Could)**

- 매칭된 동물의 `animal_age`가 만 10세 초과 → 보험 카드 대신 안내 문구: "이 아이는 신규 가입 연령(만 10세)을 넘었어요. 대신 ○○ 방법이 있어요" (차단하지 않는 톤 원칙 유지)
- MVP에서는 생략 가능. 시연 동물을 젊은 개체로 고르면 이슈 없음.

**필수 고지 (카드 하단 작은 글씨)**

- "가입 전부터 있던 질병(기왕증)은 보장에서 제외될 수 있어요" (`meta.conclusion` 반영)
- "보험료는 견종·나이·플랜에 따라 달라져요. 각 사 공식 페이지에서 확인하세요" (`meta.disclaimer` 반영)

## 5. 블록 C — 전문가 연결 (리드 제너레이션)

**UI 형태**: 카드 2장 (MVP). 조사 결론대로 병원 "예약"은 뺍니다(펫닥 예약 종료, 전용 플랫폼 부재).

| 카드 | 내용 | 연결처 (조사 표 2) | 데이터 근거 |
|---|---|---|---|
| 행동교정·훈련 | "짖음·분리불안, 혼자 고민하지 마세요. 전문 훈련사를 찾아드려요" | 숨고(훈련사 매칭) 또는 와요(방문 훈련 예약) — MVP는 링크 1개만 | 파양 1위 원인 = 행동 문제 42.7% |
| 건강검진 키트 | "병원 가기 전, 집에서 먼저 기초 건강 체크" | 핏펫 '어헤드' 키트 | 입양 전제조건 1위 = 건강 확인 35.3% |

- 브리딩(AI 교육)은 Could — 카드 3번째 자리로 여유 시 추가.
- 각 카드에 "제휴 예정" 배지 (아직 계약 전이므로 시연에서도 정직하게 표기).

## 6. 블록 D — 적응 가이드 (수익 없음, 신뢰용)

기존 Could 기능 "입양 후 맞춤 가이드북"의 축약판. "첫 7일 적응법" 정적 텍스트 1편만 배치합니다. **커머스 블록 사이에 순수 콘텐츠가 있어야 페이지 전체가 광고판으로 보이지 않습니다.** 비개발 조원이 이미 초안 작성 예정(Day 2 로드맵)이므로 추가 부담 없음.

## 7. 법적·신뢰 유의사항

1. **쿠팡 파트너스 고지 의무**: 파트너스 링크가 있는 페이지에는 "이 포스팅은 쿠팡 파트너스 활동의 일환으로, 이에 따른 일정액의 수수료를 제공받습니다" 류의 문구가 필수. 페이지 하단 고정.
2. **보험은 "정보 제공"까지만**: 보험료 비교·링크 연결은 가능하지만, 가입 권유·상담으로 보이면 모집 행위(GA 등록 필요 영역)가 될 수 있음. CTA는 "자세히 보기"(각 사 공식 페이지 이동)로 통일하고, "가입하기" 같은 문구는 쓰지 않음.
3. **가격 정보 시효**: 보험료·할인율은 `verified_date`(2026-07-02) 기준. 발표 전날 각 사 공식 계산기에서 숫자 재확인 (조사 disclaimer 그대로).

## 8. 구현 범위와 담당

| 우선순위 | 작업 | 담당 |
|---|---|---|
| Must | 4블록 정적 페이지(`/care`), JSON 2개 하드코딩 렌더링 | 개발 조원 |
| Must | 하단 제휴 고지 문구 | 개발 조원 (문구는 조장 확정) |
| Should | 진단 입력값 기반 "추천" 배지 (3절 규칙표) | 개발 조원, Must 완료 후 |
| Should | 블록 D 적응 가이드 텍스트 | 비개발 조원 |
| Could | 보험 나이 제한 개인화, 브리딩 카드 | 여유 시 |

**개발 메모**: 두 JSON을 `care_data.json` 하나로 합쳐 `insurances / checklist / services` 3개 배열로 정리 후 각 블록에서 `map()` 렌더링. 기존에 전달한 `care_tab_data_sample.json` 구조와 동일하므로 그 틀에 실데이터만 끼우면 됨.

**시연 포인트 (조장용)**: "입양 문의까지 전부 무료입니다. 그리고 입양이 결정된 순간, 이 화면에서 수익이 시작됩니다 — 용품 수수료, 보험 제휴, 전문가 연결. 특히 보험 3사는 유기동물 입양 할인이 실제로 존재해서, 저희 사용자에게는 혜택이고 저희에게는 제휴 협상 카드입니다."
