# Pawinhand 개발 지침 (Antigravity용)

> 이 스킬은 **AGENTS.md가 정의한 "무엇을(What)"을 실제로 "어떻게(How)" 코드로 옮기는지**를 담은 실무 지침입니다.
> 프로젝트 배경·기능 우선순위·데이터 정의는 `AGENTS.md`에 있으니, 그 문서를 함께 참고하세요.
> 이 문서는 **코드를 짜는 순간의 규칙**에 집중합니다.

---

## 0. 작업 전 항상 지킬 3가지

1. **비개발자 팀원이 본다는 전제.** 코드를 낼 때는 "이게 뭘 하는지 / 왜 이렇게 짰는지"를 한두 문장으로 함께 설명한다. 전문 용어는 쉬운 말로 풀어준다.
2. **4일 MVP.** "완벽한 구조"보다 "지금 가장 빠르고 검증된 방법"을 고른다. 불필요한 추상화, 과한 테스트, 조기 최적화는 하지 않는다.
3. **정해지지 않은 규칙은 단정하지 않는다.** 폴더 구조·컨벤션·API 명세 중 이 문서에 없는 것은, 임의로 확정하지 말고 사용자에게 먼저 확인한다.

---

## 1. 기술 스택 (고정 — 바꾸지 말 것)

| 구분 | 값 | 절대 규칙 |
|---|---|---|
| Frontend | Next.js | 패키지 관리는 **npm만** (yarn 금지) |
| Backend | FastAPI (Python 3.12) | 패키지 관리는 **Poetry** (`pyproject.toml`이 기준, 가상환경은 `.venv`) |
| LLM | OpenAI **`gpt-5.4-mini`** | 모델명은 **딱 한 곳(설정/상수)에서만** 정의. 하드코딩 산재 금지 |
| 오케스트레이션 | LangChain | 프롬프트 조립 → LLM 호출 → 파싱 흐름을 체인으로 묶음 |
| RAG | 벡터DB 없음 | 참고자료는 **프롬프트에 텍스트로 직접 삽입**. Chroma 등 도입 금지 |
| 배포 | Railway | API 키는 환경변수로만 |

> **왜 모델명을 한 곳에만?** 시연 중 답변 품질이 아쉬우면 `gpt-5.4`(풀 버전)로 한 줄만 바꿔 교체할 수 있어야 하기 때문입니다.

## 2. 프론트엔드 규칙 (Next.js)

### 2-1. 레이아웃은 flex + gap 위주
- 정렬은 `display: flex`로, 요소 사이 여백은 부모의 `gap`으로 잡는다.
- `margin`을 여백 용도로 남발하지 않는다. (특히 형제 요소 사이 간격은 `gap`으로.)
- 이유: margin은 상쇄(collapse)·중첩 때문에 비개발자가 디버깅하기 어렵다. `flex`+`gap`이 예측 가능하고 화면 구조가 눈에 보인다.

```jsx
// 권장: gap으로 간격을 준다
<div style={{ display: "flex", flexDirection: "column", gap: 16 }}>
  <Card />
  <Card />
</div>
```

### 2-2. 결과 화면 문구 = 사용자 언어
기술 지표·부정 단어를 그대로 노출하지 않는다.
- ❌ "활동량 지수 3/5" → ✅ "하루 30분 산책이면 충분한 아이예요"
- ❌ "부적합" / "불가" → ✅ "지금은 준비가 조금 더 필요해요"
- 척도 필드(1~5)는 숫자 대신 "활동량 보통", "사람을 좋아함"처럼 변환해 표시.

### 2-3. 상세 페이지는 재사용
- 화면 #3(유기동물 상세)과 #7(매칭 상세)은 **같은 컴포넌트**를 쓴다.
- #7에만 "왜 추천하는지"(추천 이유 + 매칭 점수) 블록이 위에 하나 더 붙는다.
- 상세 페이지를 두 개 따로 만들지 말고, `recommendReason` / `matchScore` 같은 prop 유무로 분기한다.

### 2-4. API 호출
- 백엔드 주소는 환경변수(`NEXT_PUBLIC_API_BASE_URL`)로 뺀다. 로컬/배포 주소를 코드에 직접 쓰지 않는다.

## 3. 백엔드 규칙 (FastAPI)

### 3-1. 데이터는 두 종류
- **고정 데이터**(동물 `Animal`, 보호소 `Shelter`): 포인핸드에서 수동 수집 → **JSON 파일 하나로 저장**해두고 읽어 쓴다. DB·스케줄러·자동 배치 만들지 않는다.
- **API 데이터**(진단/매칭/질문지): 요청 때마다 오가는 값. **서버에 저장하지 않는다.**

> 필드 정의(Animal 24개 필드, Shelter 필드)는 `AGENTS.md`의 "백엔드가 만들어야 하는 데이터" 표를 그대로 따른다. 여기서 재정의하지 않는다.

### 3-2. 핵심 엔드포인트 (3개)
| 메서드·경로 | 역할 | 응답 핵심 |
|---|---|---|
| `POST /api/diagnose` | 적합도 진단 | `grade`(3단계) + 잘맞는점 + 보완점 + 예상양육비 |
| `POST /api/match` | 매칭 추천 | `results[]` = `{animal_id, match_score, recommend_reason}` 3~5개 |
| `POST /api/questions` | 보호소 질문지 | 공통 템플릿(고정) 우선, AI 커스텀은 Should |

- **매칭 응답은 가볍게**: 동물 전체 정보를 담지 말고 `animal_id` + 점수 + 이유만 반환. 프론트가 `animal_id`로 고정 동물 데이터를 찾아 카드를 그린다.
- `grade`는 반드시 `"입양 가능"` / `"조건부 가능"` / `"신중히 재고"` 셋 중 하나. **"부적합"류 값 금지.**

### 3-3. 요청/응답 스키마는 Pydantic으로 고정
- FastAPI의 Pydantic 모델로 요청·응답 형태를 명시한다. (타입이 자동 검증되어 비개발자도 에러 원인을 알기 쉽다.)
- 진단·매칭의 요청 body는 동일하다(사용자 입력 2단계). 재사용한다.

### 3-4. 확정된 백엔드 폴더 구조 & 실행법 (Day 3 반영)
> **파일명 접미사로 담당을 표시합니다: 개발자 A = `_A`, 개발자 B = `_B`.** 공통 인프라(`config.py`·`main.py`)는 접미사 없이 둡니다.
> 현재는 **개발자 B 파트(진단·매칭)만** 구현되어 있습니다.

```
backend/
├── app/
│   ├── config.py            # [공통] 경로·CORS·모델명(gpt-5.4-mini) 단일 관리
│   ├── schemas_B.py         # [B] SurveyInput·DiagnoseResponse·MatchResponse
│   ├── rag_B.py             # [B] pet_adoption_rules 로더 + animals 로더
│   ├── services/
│   │   ├── ai_diagnose_B.py # [B] 진단 체인 + 폴백
│   │   └── ai_matching_B.py # [B] 매칭 체인 + 로컬 점수 폴백
│   └── routers/
│       ├── diagnose_B.py    # [B] POST /api/diagnose
│       ├── match_B.py       # [B] POST /api/match
│       ├── animals_A.py     # [A] GET /api/animals·/api/shelters  ← 담당자가 추가할 자리
│       └── questions_A.py   # [A] POST /api/questions              ← 담당자가 추가할 자리
└── main.py                  # [공통] 앱 생성·CORS·에러핸들러·라우터 등록
```

- **데이터 폴더**: RAG jsonl·`animals.json`은 **저장소 루트 `data/`** 에 둔다. (`backend/data/`가 아님 — `config.DATA_DIR`이 루트 `data/`를 가리킴)
- **모델명·경로·CORS는 `app/config.py`에서만** 정의한다. 다른 파일에서 `"gpt-5.4-mini"`를 하드코딩하지 말고 `config.OPENAI_MODEL`을 import.
- **새 라우터는 `app/routers/`에 `_A`/`_B` 접미사로 만들고** `main.py`에서 `include_router`로 등록한다. (개발자 A용 등록부는 `main.py`에 주석으로 준비돼 있음.)
- **실행**: `backend/`에서 `uvicorn main:app --reload` → 문서 `http://localhost:8000/docs`.
- **폴백 규칙**: 진단·매칭 서비스는 `OPENAI_API_KEY`가 없거나 LLM 호출이 실패하면 **로컬 규칙 결과**로 자동 폴백한다 → 키가 없어도 데모가 동작한다.

**개발자 B가 이미 구현한 엔드포인트**

| 메서드·경로 | 설명 | 응답 |
|---|---|---|
| `POST /api/diagnose` | 적합도 진단 | `{grade, good_points[], check_points[], monthly_cost}` |
| `POST /api/match` | 매칭 추천 3~5 | `{results:[{animal_id, match_score, recommend_reason}]}` |

## 4. AI / LangChain 규칙

### 4-1. 체인은 용도별로 분리
- **진단 체인**과 **매칭 체인**을 분리한다. (프롬프트·출력 형식이 다르므로 섞지 않는다.)
- 각 체인 = 프롬프트 템플릿 + LLM(`gpt-5.4-mini`) + 출력 파서.

### 4-2. 출력은 JSON 강제 (Structured Outputs)
- LLM이 "음, 결론은..." 같은 군더더기를 붙이면 파싱이 깨진다. 반드시 **JSON 스키마로 출력을 강제**한다.
- 파싱 실패에 대비해 try/except로 감싸고, 실패 시 사용자에게 "잠시 후 다시 시도" 수준의 안전한 응답을 준다.

### 4-3. RAG = 프롬프트에 텍스트 삽입
- 참고자료(파양 사유 통계·입양 전제조건·견종 특성, A4 1~2장 분량)를 **프롬프트 앞에 통째로 붙인다.**
- 벡터DB·임베딩·유사도 검색을 만들지 않는다. (분량이 작아 불필요하고, 4일 안에 디버깅 부담만 커진다.)

### 4-4. 프롬프트 설계 포인트
- 진단: 입력 2단계(생활환경 6개 + 성향 3항목)를 넣고 3단계 등급 + 근거를 받는다.
- 매칭: 사용자 입력 + 동물의 `activity`·`aggression`·`sociability`·`health_state` 등을 함께 넣어 점수·이유를 산출.
- 결과 문장은 **사용자 언어로** 생성하도록 프롬프트에 지시한다(2-2 참고).

## 5. 구현 순서 (우선순위)

**Must가 다 돌아가기 전에 Could에 손대지 않는다.**

1. (Must) FastAPI 골격 + 고정 동물/보호소 JSON 로드
2. (Must) 진단 체인 → `POST /api/diagnose` → 화면 #5(진단 결과)
3. (Must) 매칭 체인 → `POST /api/match` → 화면 #6(매칭 목록)
4. (Must) 화면 #3/#7 상세(재사용) + #2 목록
5. (Must) 화면 #8 질문지(공통 템플릿 고정)
6. (Should) 체크리스트·입양 문의 안내·AI 커스텀 질문
7. (Could) 검색/필터, 화면 #4 지도 + 클러스터링

## 6. 막혔을 때 (에러 대비 가이드)

비개발자 팀원이 자주 만날 상황을 미리 안내한다.

| 증상 | 이런 메시지가 뜨면 | 이렇게 |
|---|---|---|
| npm 설치 실패 | `ERR! peer dep` 등 | Node 버전 확인, `npm install` 재시도, 그래도 안 되면 에러 전문을 그대로 공유 |
| Poetry 인식 안 됨 | `command not found: poetry` | Poetry 설치 여부 확인, `.venv` 경로 확인 |
| API 키 오류 | `401 Unauthorized` (OpenAI) | `.env`(로컬)·Railway 환경변수에 키가 있는지 확인. 키를 코드/Git에 넣지 말 것 |
| LLM 응답 파싱 실패 | JSON 파싱 에러 | 출력 JSON 강제(4-2)가 걸려 있는지 확인, try/except로 감싸기 |
| CORS 에러 | 프론트에서 API 호출 시 `blocked by CORS` | FastAPI에 CORS 미들웨어 설정(프론트 주소 허용) |

> 에러가 나면 **메시지 전문을 그대로 복사해서** 공유하도록 안내한다. 추측으로 넘겨짚지 않는다.

## 7. 보안 / 데이터 원칙 (짧게)

- 진단·매칭 요청·응답은 **서버에 저장하지 않는다.**
- 사용자 입력은 개인식별정보가 아니므로 **회원가입 없이 익명** 수집. 이름·연락처·주소는 아예 받지 않는다.
- OpenAI API 키는 `.env`(로컬) / Railway 환경변수(배포)로만. **Git에 커밋 금지.**

> **협업/Git 규칙(브랜치·커밋·PR·의존성 lock 파일 커밋)은 `AGENTS.md`의 "협업 / Git 규칙" 섹션을 따릅니다.** 특히 `main` 직접 푸시 금지, 패키지 추가 시 lock 파일(`package-lock.json` / `poetry.lock`) 동시 커밋을 반드시 지킬 것.
